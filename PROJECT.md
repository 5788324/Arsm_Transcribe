# Arsm Transcribe 项目文档

> 本文档与 `WORKLOG.md` 是 AI 协作的权威入口；`README.md` 是面向使用者的公开说明。接手项目必须先完整阅读本文档，再阅读 WORKLOG 顶部最新记录。

## 0. 给 AI 的工作规则

1. 开工前完整阅读 `PROJECT.md` 和 `WORKLOG.md` 最新记录。
2. 改模块前先在 WORKLOG 顶部写“打算做什么、为什么”。
3. 会话结束前必须补充实际改动、踩坑和下一步。
4. 不确定的模型行为、VAD 效果或字幕语言不得猜测，标记“待人工确认”。
5. 修改功能后同步更新本文档状态表和 README。
6. **阶段模块只能通过磁盘 JSON 通信**；SQLite 只能保存资料库索引和任务状态，不能代替 raw/clean/translated JSON。

## 1. 产品目标

面向 Windows 本地 RJ/ASMR 音声库，将资料库中的每个媒体版本处理为播放器可加载的中文字幕：

```text
音频或已有字幕
  → 来源识别与处理计划
  → ASR（需要时）
  → 分段清洗
  → 本地模型翻译（需要时）
  → 同名中文 .lrc + 中文 .vtt
```

默认资料库为 `E:\arsm`，后续下载继续放入该目录，由用户手动点击“扫描新文件”。每个 WAV、MP3、有音效、无音效版本均视为独立媒体，不按 stem 去重。

### 产品边界

- 音频与字幕内容全程本地处理，不上传云端。
- 首版不做实时转录、说话人分离或模型下载器。
- TXT/PDF 仅作为台词、角色名和专有名词校对参考，没有逐句时间轴时不能直接生成 LRC。
- 项目独立于 Yang-Kura；未来只通过版本化命令行 JSON 接口集成，不共享数据库。

## 2. 核心架构和数据

### 2.1 阶段产物

```text
cache/*.transcript.raw.json
cache/*.transcript.clean.json
cache/*.translated.json
音频目录/audio.lrc
音频目录/audio.vtt
```

转录是最昂贵步骤。更换翻译模型时只重跑翻译和字幕生成，不重跑 ASR。所有 JSON 写入和最终字幕写入应使用临时文件完成后原子替换。

### 2.2 引擎接口

公开 CLI：

- `scan ROOT...`：更新 SQLite 增量资料库，并输出 `api_version=1.0` 的扫描 JSON。
- `plan ROOT...`：只生成处理计划，不处理音频、不改写音频目录。
- `doctor`：检查 ASR 执行器、缓存、数据库目录和翻译服务。
- status：读取当前批次状态。
- cancel：请求在当前文件完成后停止批次，保留已完成阶段结果。
- `run-single`、`run-batch`、`retry-failed`：执行处理链路。
- `transcribe`、`clean`、`translate`、`write-lrc`：分阶段调试；`write-lrc` 当前同时生成 LRC/VTT。

SQLite 默认位于 `data/library.db`，保存路径、大小、mtime、处理动作、来源字幕和最后扫描时间。模块内容仍通过磁盘 JSON 传递。

## 3. 已验证的 ASR 格式与清洗规则

ChickenRice/cu128-transcribe 的真实 LRC 输出示例：

```text
[00:03.04]い、いらっしゃい。待ってたよ……
[00:06.86]
[00:08.38]どう?この日のために用意した、特別仕様の巫女服。
```

空文本时间戳代表上一句结束时间，不是独立台词。解析器必须合并到上一 segment 的 `end`。相邻句超过配置阈值的长空档只标记为疑似漏识别，不自动编造内容；是否为 VAD 漏识别仍需人工试听确认。

清洗阶段必须独立存在：短间隔碎片在不超过单行上限时合并；超长句按标点拆分并分配时间区间；长静音写入质量标记。

## 4. 已有字幕策略

匹配优先级包含真实库格式：`audio.wav.vtt` / `audio.mp3.vtt`、`audio.wav.srt`、日文 LRC、普通 `audio.vtt/srt`。

| 来源 | 默认动作 |
|---|---|
| 本项目或已有中文 LRC | 跳过 |
| 中文/非日文 VTT、SRT | 保留时间轴，直接生成中文 LRC/VTT，不调用翻译 |
| 日文 VTT、SRT、LRC | 保留时间轴，只翻译文本 |
| 语言未知定时字幕 | 待人工确认，不覆盖 |
| 无定时字幕 | ASR → 清洗 → 翻译 |
| TXT/PDF | 记录为参考文本，当前不直接处理 |

默认输出为 `audio.lrc` 和 `audio.vtt`，包含 `by yang 创建`声明。同名来源文件不得静默覆盖；VTT 冲突时生成 `audio.zh.vtt`。LRC 同名来源冲突目前由策略跳过或翻译路径处理，后续 UI 提供明确替换操作。

## 5. 批处理鲁棒性

- 断点续跑：有效阶段 JSON 和输出存在时默认跳过，重跑必须显式 `--overwrite`。
- 单文件失败记录到 `logs/failed.txt`，其余任务继续。
- `retry-failed` 只重试失败项并备份旧清单。
- 进度写入 `logs/batch_status.json`，桌面端可读取。
- 支持中文、日文、特殊字符和超长路径；终端编码异常不得污染 JSON。
- 大库执行前先 `plan` 预览，未经用户确认不对 `E:\arsm` 全量写入。

## 6. 配置和可替换模型

ASR backend、执行器路径、模型目录、VAD 参数、本地 OpenAI-compatible 地址、模型标识、批大小、超时和 prompt 全部来自 `config.yaml`。当前实测 ASR 为 `F:\AI\Model\models\infer.exe`；Qwen3-ASR 仍是占位，未来必须配 ForcedAligner 才能产生逐句时间轴。

翻译服务可使用 LM Studio/Ollama。模型可随时替换，模型质量必须通过固定样本 A/B 对比，不因接口可调用就判定质量合格。

## 8. 已知风险和待人工确认

- 当前 VAD 阈值为 0.5，文档早期建议 0.2；哪个更适合 ASMR 必须试听对比。
- LM Studio 曾白屏和服务不可用；2026-07-13 `doctor` 再次检测到接口可用，但模型质量与长批稳定性仍需实测。
- 语言判断为字符启发式，未知结果必须人工确认。
- 同一作品中来源字幕可能错配；只有完整文件名或明确同 stem 才自动复用。
- PySide6 当前环境未安装，正式 UI 尚未开始，不得把 tkinter 过渡界面描述为最终产品。
